This section shows the following VisionX usage scenarios with EDB Postgres Advanced Server and EDB Postgres Extended.

1. Export an Oracle based VisionX application and import the application to now use Postgres Advanced Server/EDB Postgres Extended instance as a data source.
2. Use VisionX to create applications that use an EDB Postgres Advanced Server/EDB Postgres Extended instance as a data source.
   - Create a Screen which uses an existing table from EDB Postgres Advanced Server/EDB Postgres Extended instance to connect to.
   - Create a Screen which creates the table into EDB Postgres Advanced Server/EDB Postgres Extended instance to connect to.

## Export and Import VisionX Application

The example shows how an Oracle based VisionX application is created, exported and then imported to use an EDB Postgres Advanced Server/EDB Postgres Extended instance as data source.

1. Open VisionX and create an application. The "Create a new Application" screen will then be opened.

</p>

2. On Choose Database, enter the connection information for Oracle.

</p>

<p align="center">
  <img  alt="Choose Database" src="Images/ImportExport3.png">
</p>

3. On Create Admin User, provide the user name and password.

<p align="center">
  <img  alt="Create Admin User" src="Images/ImportExport4.png">
</p>

4. A Task List screen will be displayed which will display the Application creation progress.

<p align="center">
  <img  alt="Task List" src="Images/ImportExport5.png">
</p>

5. Once the application is created then New Application Screen will be opened which will allow the user to create a screen which will be connected to the Oracle Table and provide the user ability to perform SELECT and DML Operations.

<p align="center">
  <img  alt="New Application Screen" src="Images/ImportExport6.png">
</p>

6. On Screen Infos, provide the required information for the screen.

<p align="center">

7. On Choose Layout, select the required layout for the screen.

<p align="center">
  <img  alt="Choose Layout" src="Images/ImportExport8.png">

8. On Select Data Source, select Define the information you want to manage option, this will allow user to define the structure of the table which will be created on Oracle.

<p align="center">
  <img  alt="Select Data Source" src="Images/ImportExport9.png">
</p>

9. On Data/Informations, create the structure of the table.

11. Once the screen is created then it will be displayed in Design Mode where user can adjust it according to his requirements. 

<p align="center">

</p>

13. To Export the Application, click on Export option. Export Application screen will be opened.

17. On Content Selection, select option All.

<p align="center">
  <img  alt="Content Selection" src="Images/ImportExport17.png">

18. On Check Datasources, enter the connection information for Postgres Advanced Server (EPAS)/EDB Postgres Extended instance which will now be used with the this Application instead of Oracle.

  <img  alt="Check Datasources" src="Images/ImportExport18.png">

<p align="center">

</p>

20. A Task List screen will be displayed which will display the Import progress.

<p align="center">

</p>

21. Once the Import is completed then a new Application will be created.

<p align="center">
  <img  alt="New Application" src="Images/ImportExport22.png">

22. Open the Application and then open the Screen in Preview mode and perform DML operations, the data now will be updated in Postgres Advanced Server (EPAS)/EDB Postgres Extended instance.
