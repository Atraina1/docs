This section shows the following VisionX usage scenarios with EDB Postgres Advanced Server and EDB Postgres Extended.

1. Export an Oracle-based VisionX application and import the application to now use EDB Postgres Advanced Server/EDB Postgres Extended instance as a data source.
2. Use VisionX to create applications that use an EDB Postgres Advanced Server/EDB Postgres Extended instance as a data source.
   - Create a Screen which uses an existing table from an EDB Postgres Advanced Server/EDB Postgres Extended instance to connect to.
   - Create a Screen which creates the table into an EDB Postgres Advanced Server/EDB Postgres Extended instance to connect to.

## Export and Import VisionX Application

The example shows how an Oracle based VisionX application is created, exported and then imported to use an EDB Postgres Advanced Server/EDB Postgres Extended instance as data source.

1. Open VisionX and create an application. The "Create a new Application" screen will then be opened.

</p>

2. On Choose Database, enter the connection information for Oracle.

</p>

<p align="center">
  <img  alt="Choose Database" src="Images/ImportExport3.png">
</p>

3. On Create Admin User, provide the user name and password.

<p align="center">
  <img  alt="Create Admin User" src="Images/ImportExport4.png">
</p>

4. A Task List screen will be displayed which will display the Application creation progress.

<p align="center">
  <img  alt="Task List" src="Images/ImportExport5.png">
</p>

5. Once the application is created then the New Application Screen will be opened which will allow the user to create a screen that will be connected to the Oracle Table and provide him/her the ability to perform SELECT and DML Operations.

<p align="center">
  <img  alt="New Application Screen" src="Images/ImportExport6.png">
</p>

6. On Screen Infos, provide the required information for the screen.

7. On Choose Layout, select the required layout for the screen.

<p align="center">
  <img  alt="Choose Layout" src="Images/ImportExport8.png">

<p align="center">
  <img  alt="Select Data Source" src="Images/ImportExport9.png">

9. On Data/Informations, create the structure of the table.

13. To export the application, click on the Export option. The Export Application screen will be opened.

17. On Content Selection, select All.

18. On Check Datasources, enter the connection information for the EDB Postgres Advanced Server/EDB Postgres Extended instance which will now be used with the application instead of Oracle.

20. The Task List screen will be displayed which will display the Import progress.

21. Once the import is completed then a new application will be created.
